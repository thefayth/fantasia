# Image Brief

## Visual Rule

Use what Fantasia really looks like. Do not invent fake product screens.

## Approved Public Visual Types

- Real desktop screenshots from the current build.
- Real COAI Assets Hub screenshots from the current build.
- Real narrow/mobile layout screenshots from the current build.
- Real brand board/deck/demo visuals when captured from actual local files.
- Public-safe workflow diagrams clearly labeled as diagrams.
- Simple branded banners that do not pretend to be product UI.

## Avoid

- Fake app dashboards.
- Invented interface states.
- API endpoint names.
- Private architecture.
- Credentials, tokens, or secrets.
- Private local paths.
- Customer data.
- Unpublished creative work.
- Screenshots with private windows or personal information.
