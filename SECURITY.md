# Security Policy

## Supported Surface

This public repository is documentation-first. It does not contain Fantasia core source code, credentials, private workflows, production adapters, or customer/project data.

## Do Not Submit Secrets

Do not open issues, pull requests, comments, emails, or discussions containing:

- API keys, app passwords, OAuth tokens, worker tokens, or signing secrets.
- WordPress, server, IONOS, Canva, OpenAI, COAI, Google, or GitHub credentials.
- Private URLs, signed URLs, local filesystem paths, customer data, medical records, family information, legal/admin records, or unpublished creative work.

If a secret is accidentally exposed, revoke it immediately in the original platform and contact the project owner through a private channel.

## Responsible Disclosure

For suspected security issues, share only the minimum public-safe summary needed to explain the risk. Do not include exploit instructions, private data, or proof materials that reveal sensitive infrastructure.

## Public Boundary

This repo intentionally avoids private source architecture and production runbooks. Security reports should respect that boundary.
